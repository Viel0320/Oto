# 2026-06-23

## Koin 全面接入完成

将 APlayer 的手动 DI 全面替换为 Koin 4.2.2。

### 变更摘要
- 删除 `AppContainer.kt`、`di/graph/` 目录（7 个 Graph 文件）
- 新增 `di/koin/` 目录（19 个 Koin module 文件 + `APlayerKoinApplication` + `GraphClosePolicy` + `ViewModelModule`）
- 6 个 singleton 类（`PlaybackManager`/`AutoRewindManager`/`AppDatabase`/`AppSettingsRepository`/`SearchHistoryStore`/`AbsCredentialStore`）改为 `internal constructor`，删除 `getInstance` 静态方法
- 11 个 ViewModel 从 `AndroidViewModel(application)` 改为 `ViewModel()` + 构造函数注入依赖视图接口
- `APlayerApp.kt` 用 `koinViewModel()` 替换手写 `ViewModelProvider.Factory`
- Service/Worker/Receiver 改为 `GlobalContext.get().get<T>()`
- `APlayerApplication.onCreate` 启动 Koin，`onTerminate` 调用 `GraphClosePolicy.closeInLifecycleOrder()` + `stopKoin()`
- `GraphClosePolicy` 保留有序关闭：media → download → abs → library → uiEvents
- `DataStore<Preferences>` 用 `named` qualifier 区分（appSettings/searchHistory/absCredentials）
- 更新 `AGENTS.md` DI 策略条款
- 更新 5 个架构测试以验证 Koin module 结构
- 修复 `SettingsHandlers.kt` 的 data layer 违规：引入 `DuplicateLibraryRootSource` application 层枚举

### 关键决策
- Koin 4.2.2 的 `koin-androidx-compose` 不含 `koinInject`，用 `GlobalContext.get().get<T>()` 替代
- ViewModel DSL 用 `org.koin.core.module.dsl.viewModel`（非 deprecated 的 `org.koin.androidx.viewmodel.dsl.viewModel`）
- Robolectric 测试中 `APlayerApplication.onCreate` 会启动完整 Koin，测试用 `loadKoinModules` 覆盖 fake
- startup warmup 协程加 `runCatching` 防止 Koin scope 关闭后抛未捕获异常
- `getInstance` 方法保留作为 Koin 桥接（委托到 `GlobalContext.get().get<T>()`），因为 library/abs 层内部类仍用 `getInstance` 获取 `AppDatabase`/`AppSettingsRepository`/`AbsCredentialStore`
- `AbsCoverCache` 默认参数中的 `getInstance` 调用已删除，改为必传参数
- `DataStore<Preferences>` 用 `named` qualifier 区分，`AbsModule` 中 `AbsCredentialStore(get(named("absCredentials")))` 必须指定 qualifier
- `AbsApiClient` 接口绑定到 `RealAbsApiClient`：`single<AbsApiClient> { get<RealAbsApiClient>() }`
- `CoverSelfHealer` 接口绑定到 `CoverRecoveryHelper`：`single<CoverSelfHealer> { get<CoverRecoveryHelper>() }`
- `GraphClosePolicy.register` 的 `single` 必须用 `single<Unit>` 且不能返回已注册的类型，否则会导致 Koin 无限递归（StackOverflowError）
- `createdAtStart = true` 会导致启动时立即解析依赖链，触发循环依赖，必须避免
